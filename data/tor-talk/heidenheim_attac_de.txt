@_subject: More flexible IP-tables and TOR ? 
It is certainly cool to ONLY allow Tor to the internet, but in my
opinion in real world, there are some connections/circumstances where
you want don't want to have Tor in the middle: Mail, Webaccounts,
data-intensive downloads/upgrades. I think, that's where p.e. Torbutton comes in ... and the trouble
starts. Being certainly *NOT* a firewall-expert, I'm using "Firestarter" to
handle my IPtables-Configuration. I configured the TransparentProxy via the Firestarter user-pre-file from
this source: I don't know why, but I assume with this option, Tor drops the connection after some time. So I set torrc: KeepalivePeriod 20
and now it works for me.
Firestarter outgoing rules (whitelist) : Allow port 80 and , 143, 443 from firewall.
I don't know if that configuration makes any sense, but it seems more flexible for me.
Improvements very welcome,
